# Samples

Only commit sanitized sample fixtures here.

Do not commit:

- real client exports unless explicitly approved and sanitized
- API keys, tokens, passwords, or connection strings
- screenshots with account IDs or private data
- PDFs, spreadsheets, or raw downloaded reports from live accounts

For real validation work, keep local-only files under ignored folders such as:

- `data/`
- `raw/`
- `exports/`
- `outputs/`

The first requested validation package should include one clean week and one messy week, with raw exports plus final manually cleaned output for comparison.
